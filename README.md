# 📱 Proyecto Mobile - Bodega Reyna

## 🧩 Definición del Problema
La bodega no cuenta con un sistema digital que permita gestionar reservas, ventas y clientes, lo cual genera desorganización y limita la experiencia del usuario.

## 👥 Usuarios Involucrados
- Clientes / Turistas
- Administradores

## 🎯 Objetivos
- Gestionar reservas
- Facilitar compras
- Mejorar la experiencia del usuario
- Optimizar procesos internos

## ⚙️ Requerimientos Funcionales
- Registro de usuarios
- Búsqueda de productos y experiencias
- Reserva de tours
- Gestión de inventario
- Notificaciones

## 🔒 Requerimientos No Funcionales
- Usabilidad (interfaz amigable)
- Seguridad (protección de datos)
- Rendimiento (respuestas rápidas)
- Disponibilidad (99.9%)

## 🧾 Historias de Usuario

- Como cliente quiero registrarme para acceder a los servicios.
- Como usuario quiero reservar tours fácilmente.
- Como cliente quiero comprar productos.
- Como administrador quiero gestionar el inventario.

## 🔗 Justificación
Las historias de usuario están alineadas con los requerimientos funcionales, ya que representan las funcionalidades principales del sistema desde la perspectiva del usuario.
